Installing Free SSL Certificate with Let's Encrypt

üìñ 6 min read üëÅÔ∏è 24.7k views üìÖ Updated Dec 2025

SSL (Secure Sockets Layer) certificates encrypt the connection between your website and your visitors, protecting sensitive information like passwords, credit card details, and personal data. SSL is essential for every modern website, and Google even uses HTTPS as a ranking factor.

This guide will show you how to install a free SSL certificate using Let's Encrypt through cPanel's built-in SSL/TLS manager.

Why You Need SSL Certificate

  • Security: Encrypts data transmission between server and browser
  • Trust: Shows padlock icon and "Secure" label in browser
  • SEO Boost: Google favors HTTPS websites in search rankings
  • Compliance: Required for e-commerce and payment processing
  • Browser Warnings: Avoid "Not Secure" warnings in modern browsers
  • Customer Confidence: Visitors trust secure websites more
üîí What is Let's Encrypt? Let's Encrypt is a free, automated, and open Certificate Authority (CA) that provides SSL certificates trusted by all major browsers. Certificates are valid for 90 days and auto-renew automatically.

Prerequisites

  • Active AxiomHost hosting account
  • Domain name pointed to your hosting (DNS propagation complete)
  • cPanel login credentials
  • Your website must be accessible via HTTP first
‚ö†Ô∏è Important: Make sure your domain DNS is fully propagated and pointing to AxiomHost servers before installing SSL. Use DNS Checker to verify propagation.

Step-by-Step: Installing SSL Certificate

Step 1: Access cPanel SSL/TLS Manager

  1. Log into your cPanel at yourdomain.com/cpanel
  2. Scroll down to the Security section
  3. Click on SSL/TLS Status
üí° Alternative Method: You can also click on "Let's Encrypt SSL" directly if available in your cPanel interface for a streamlined installation process.

Step 2: Select Domains for SSL

  1. You'll see a list of all domains and subdomains in your account
  2. Check the boxes next to domains you want to secure with SSL
  3. Recommended: Select both yourdomain.com and www.yourdomain.com
üìå Domain Selection Tips:
  • Select your main domain AND www subdomain
  • Select mail.yourdomain.com for webmail SSL
  • Select any subdomains you actively use
  • You can install SSL for multiple domains at once

Step 3: Install SSL Certificate

  1. After selecting domains, click Run AutoSSL button
  2. cPanel will automatically:
    • Request SSL certificate from Let's Encrypt
    • Verify domain ownership
    • Install the certificate
    • Configure auto-renewal
  3. Wait 1-2 minutes for the process to complete
üí° Installation Time: SSL installation typically takes 1-2 minutes. If it takes longer than 5 minutes, refresh the page and check the status.

Step 4: Verify SSL Installation

  1. Once complete, you'll see "SSL certificate successfully installed" message
  2. The status will change to a green checkmark with "AutoSSL" label
  3. Visit your website using https://yourdomain.com
  4. Look for the padlock icon üîí in the address bar
‚úÖ Success Indicators:
  • Padlock icon appears in browser address bar
  • "Connection is secure" when clicking padlock
  • HTTPS in the URL instead of HTTP
  • No browser security warnings

Step 5: Force HTTPS (Redirect HTTP to HTTPS)

After installing SSL, force all traffic to use HTTPS to ensure maximum security.

Method 1: Using cPanel (Easiest)

  1. In cPanel, go to Domains section
  2. Click Domains
  3. Find your domain and toggle Force HTTPS Redirect to ON
  4. Click Save

Method 2: Using .htaccess (Advanced)

Add this code to your .htaccess file in the root directory:

RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
üí° Recommended: Use Method 1 (cPanel toggle) as it's simpler and less prone to errors. Only use .htaccess if you need advanced configuration.

Method 3: Force HTTPS in WordPress

If you're running WordPress, update your site URL:

  1. Log into WordPress admin panel
  2. Go to Settings > General
  3. Change both URLs from http:// to https://:
    • WordPress Address (URL)
    • Site Address (URL)
  4. Click Save Changes
  5. You'll be logged out - log back in using HTTPS
‚ö†Ô∏è WordPress Warning: Make sure SSL is properly installed before changing WordPress URLs. Changing to HTTPS without a valid certificate will lock you out of your site.

Automatic SSL Renewal

Let's Encrypt certificates are valid for 90 days. AxiomHost's cPanel automatically renews SSL certificates before expiration.

How Auto-Renewal Works

  • AutoSSL runs daily at midnight server time
  • Checks all certificates expiring within 30 days
  • Automatically renews expiring certificates
  • No manual intervention required
  • Email notification sent if renewal fails
üí° Peace of Mind: You don't need to worry about SSL expiration. Our system handles renewals automatically, ensuring your site stays secure 24/7.

Verifying SSL Certificate Details

To view your SSL certificate information:

  1. Visit your website via HTTPS
  2. Click the padlock icon in the address bar
  3. Click Certificate or Certificate is valid
  4. View certificate details:
    • Issued to: yourdomain.com
    • Issued by: Let's Encrypt
    • Valid from: Issue date
    • Valid to: Expiration date (90 days)

Troubleshooting Common SSL Issues

Issue: "Not Secure" Warning Still Showing

Causes and Solutions:

  • Mixed Content: Some resources loading via HTTP
    • Solution: Update all links to use HTTPS or relative URLs
    • Check browser console (F12) for mixed content warnings
  • Browser Cache: Old HTTP version cached
    • Solution: Clear browser cache and hard refresh (Ctrl+Shift+R)
  • Wrong URL: Accessing HTTP instead of HTTPS
    • Solution: Type https:// manually or force HTTPS redirect

Issue: AutoSSL Failed

Common Reasons:

  • DNS not pointing to AxiomHost
  • Domain not accessible via HTTP
  • .htaccess blocking Let's Encrypt validation
  • Cloudflare or other proxy in the way

Solutions:

  1. Verify DNS is correctly pointed to AxiomHost nameservers
  2. Temporarily disable .htaccess password protection
  3. If using Cloudflare, set SSL mode to "Full" not "Flexible"
  4. Contact AxiomHost support for manual installation

Issue: SSL Certificate Expired

This should never happen with AutoSSL, but if it does:

  1. Go to cPanel > SSL/TLS Status
  2. Select the domain with expired certificate
  3. Click Run AutoSSL to request new certificate
  4. If problem persists, contact AxiomHost support

Issue: Wildcard SSL Needed

Free Let's Encrypt SSL covers your domain and www subdomain. For unlimited subdomains:

  • Contact AxiomHost support for wildcard SSL installation
  • Or purchase commercial wildcard SSL certificate
  • Wildcard format: *.yourdomain.com

SSL Best Practices

Security

  • Always force HTTPS redirect for all pages
  • Enable HSTS (HTTP Strict Transport Security)
  • Use strong TLS protocols (TLS 1.2 and 1.3)
  • Regularly scan for mixed content issues
  • Keep WordPress/CMS updated to support latest SSL standards

Performance

  • SSL adds minimal overhead to page load
  • HTTP/2 requires HTTPS and improves performance
  • Use CDN with SSL support for global speed
  • Enable OCSP stapling for faster certificate validation

Maintenance

  • Monitor SSL certificate expiration dates
  • Check AutoSSL logs regularly in cPanel
  • Test SSL configuration using SSL Labs
  • Keep admin email updated for renewal notifications

Testing Your SSL Installation

Manual Browser Test

  1. Visit https://yourdomain.com
  2. Look for padlock icon in address bar
  3. Click padlock and verify certificate is valid
  4. Check for any mixed content warnings in browser console (F12)

Online SSL Checker

Use these free tools to verify your SSL setup:

  • SSL Labs: https://www.ssllabs.com/ssltest/
    • Comprehensive SSL/TLS analysis
    • Grades your SSL configuration (aim for A+)
    • Identifies security vulnerabilities
  • Why No Padlock: https://www.whynopadlock.com/
    • Finds mixed content issues
    • Shows which resources are insecure
